What Is MobileApplication Schema?

MobileApplication Schema is a subtype of SoftwareApplication that describes a mobile app designed for smartphones and tablets. It includes properties such as name, description, URL, image, operating system (iOS, Android), application category, pricing, file size, and aggregate ratings. Google uses this markup to display app rich results in search, helping users discover and compare mobile applications.

For app developers, mobile marketing teams, and app review sites, MobileApplication Schema is essential for visibility in both Google Search and app store indexing.

Features

  • Complete app details: name, description, URL, and image
  • Operating system selection (iOS, Android, or both)
  • Application category for proper classification
  • Pricing with currency and availability status
  • File size in MB for download information
  • Author/publisher organization details
  • Aggregate rating with rating value and review count

Frequently Asked Questions

Can I use MobileApplication Schema for both iOS and Android on one page?

Yes, use separate schema blocks or specify multiple operating systems for a single app entry.

Is this different from SoftwareApplication Schema?

MobileApplication is a subtype of SoftwareApplication specifically optimized for mobile apps with mobile-focused properties.

Do I need to include the app store URL?

Yes, linking to your app store page helps users find and download your app directly.
