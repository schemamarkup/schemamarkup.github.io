How do I create Schema Markup?

June 2026 · 572 words

Creating schema markup is easier than most people think. There are several methods available, ranging from manual coding to using automated tools. Choose the method that best matches your technical skills and the complexity of your needs.

Method 1: Using a Schema Markup Generator (Easiest)

The easiest way to create schema markup is to use a schema markup generator tool like our Schema Markup Generator. Simply select the schema type you need, fill in the form fields with your content, and the tool generates the JSON-LD code automatically. You can then copy and paste the code into your website. This method requires no coding knowledge and takes just a few minutes.

Method 2: Writing JSON-LD Manually

If you prefer to write schema markup yourself, JSON-LD is the recommended format. Start with the @context property set to https://schema.org, then add the @type property with the schema type you want to use. Add the relevant properties for that type with your content values. Here is a simple example of Organization schema in JSON-LD format: a script tag with type application/ld+json containing the JSON object with your schema properties.

Method 3: Using CMS Plugins

If you use a content management system like WordPress, there are plugins that can generate schema markup automatically. Yoast SEO, Rank Math, and Schema Pro are popular options. These plugins often provide default schema markup for common content types and allow you to customize the markup for specific pages. This method is ideal for non-technical users who want a set-and-forget solution.

Understanding JSON-LD Structure

JSON-LD schema markup follows a specific structure. The @context property tells search engines which vocabulary you're using (always https://schema.org). The @type property specifies the schema type. Additional properties describe your content. Properties can be simple values like strings and numbers, or they can be nested objects for complex relationships. For example, an Offer might be nested within a Product.

Required vs. Recommended Properties

Each schema type has required properties that must be present for the markup to be valid. For example, Product schema requires name. Some properties are recommended for better rich result eligibility but are not strictly required. Check Google's developer documentation for the specific requirements of each schema type. Missing required properties can prevent your markup from generating rich results.

Testing Your Markup

After creating your schema markup, always test it before adding it to your live site. Use Google's Rich Results Test to check if your markup is valid and eligible for rich results. The tool will identify any errors or warnings and show you how your rich result might appear in search. Fix any issues before deploying the markup to your website.

Adding to Your Website

Once your markup is created and tested, add it to your website. For JSON-LD, place the script tag in the head section of your HTML page. For Microdata and RDFa, the markup is added directly to your visible HTML elements. Make sure the markup is on the same page as the content it describes. After adding the markup, use the Rich Results Test again to confirm it's working correctly on your live page.

Monitoring Performance

After implementing schema markup, monitor its performance using Google Search Console. The Structured Data report shows you which pages have markup, how many rich result impressions they're getting, and how many clicks they receive. This data helps you understand the impact of your schema markup and identify any issues that need fixing.
