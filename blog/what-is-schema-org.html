What is Schema.org? The Foundation of Structured Data

Schema.org is a collaborative community activity that creates, maintains, and promotes schemas for structured data on the internet. Founded in 2011 by Google, Microsoft, Yahoo, and Yandex, Schema.org has become the de facto standard for structured data markup on the web.

The Purpose of Schema.org

The primary goal of Schema.org is to create a shared vocabulary that webmasters can use to mark up their pages in ways that major search engines understand. Before Schema.org, each search engine had its own approach to structured data, creating confusion and extra work for webmasters.

By providing a unified vocabulary, Schema.org simplified the process of implementing structured data. Today, the Schema.org vocabulary includes hundreds of types and properties covering everything from creative works to events to people and organizations.

How Schema.org Works

Schema.org provides a hierarchical taxonomy of types. Each type has specific properties that describe its characteristics. For example, the Product type includes properties like name, description, brand, offers, and aggregateRating.

Types can inherit properties from their parent types. The LocalBusiness type, for instance, inherits from Place and Organization, giving it access to properties like address, telephone, and opening hours, in addition to its own specific properties.

Key Schema.org Types

Some of the most widely used Schema.org types include Article, Product, LocalBusiness, Event, Recipe, Review, FAQPage, VideoObject, and Person. Each type serves a specific purpose and enables specific rich result features in Google Search.

The Schema.org vocabulary continues to evolve with new types and properties added regularly to reflect changes in technology and search engine capabilities.
