How do I add Schema Markup to my website?

June 2026 · 576 words

Adding schema markup to your website can be done in several ways depending on your technical expertise and the platform you use. The method you choose will depend on whether you use a content management system, have access to your site's HTML files, or prefer a code-free solution.

Method 1: Adding JSON-LD to HTML

JSON-LD is the recommended format by Google. To add it, create a script tag with type application/ld+json and place your schema markup inside. Add this script tag to the head section of your HTML page, or just before the closing body tag. The markup should be on the same page as the content it describes. Here is the basic structure: a script tag containing your JSON-LD object with @context and @type properties.

Method 2: Using WordPress Plugins

WordPress users can add schema markup without touching code by using SEO plugins. Yoast SEO automatically adds schema markup for articles, pages, and other content types. Rank Math provides more detailed schema control with built-in templates for common types. Schema Pro allows you to create custom schema markup for specific pages and post types. These plugins handle the technical implementation so you can focus on content.

Method 3: Using a Tag Manager

Google Tag Manager can be used to add schema markup without editing your website's HTML directly. Create a Custom HTML tag in GTM, paste your JSON-LD code into the tag, and set it to fire on the pages where you want the schema to appear. This method is useful for websites where you don't have direct access to the HTML files or want to manage markup centrally.

Method 4: Adding to Specific CMS Platforms

For Shopify, add schema markup to your theme's liquid files or use a schema app from the Shopify App Store. For Squarespace and Wix, use the built-in settings panels or code injection features to add JSON-LD. For Joomla and Drupal, use SEO extensions that support schema markup. Each platform has specific methods for adding custom code to your pages.

Where to Place Schema Markup

JSON-LD can be placed in the head or body of your HTML page. Google's documentation says either location is acceptable. For Microdata and RDFa, the markup must be added directly to the visible HTML elements that contain your content. This makes JSON-LD easier to implement because it's separate from your visible content and won't affect your page layout.

Adding Multiple Schema Types

You can add multiple schema types to a single page by including multiple JSON-LD script blocks or by nesting types within a single script using @graph. For example, a product page might include Product schema, Offer schema, and Review schema together. Each type describes a different aspect of your content and can enable different rich result features.

Testing After Adding

After adding schema markup to your website, test it using Google's Rich Results Test. Enter your page URL and the tool will scan for schema markup and report any issues. Fix any errors or warnings found. You can also use the Schema Markup Validator for a more detailed analysis of your markup syntax.

Monitoring in Search Console

Once your schema markup is live, monitor it in Google Search Console. The Enhancements section shows reports for each type of rich result you're targeting. These reports show how many pages have valid markup, how many impressions your rich results receive, and any errors that need attention. Regular monitoring ensures your markup continues to work correctly.
