How do I test Schema Markup?

June 2026 · 538 words

Testing schema markup is a critical step in the implementation process. Even small errors in your markup can prevent Google from generating rich results. Fortunately, Google provides free tools that make testing quick and straightforward.

Google Rich Results Test

The Rich Results Test is Google's primary tool for testing schema markup. You can test either a URL or paste your markup code directly. The tool checks your markup against Google's rich result requirements and shows which rich result types your page is eligible for. It also provides a preview of how your rich result might appear in search results. Any errors or warnings are clearly highlighted with explanations and suggestions for fixing them.

Google Schema Markup Validator

The Schema Markup Validator is a more comprehensive testing tool that validates all Schema.org types, not just those that produce rich results. It checks your markup syntax, verifies that all properties are valid for the specified type, and identifies any missing required properties. This tool is useful for comprehensive testing of complex markup with multiple schema types.

Testing with Search Console

Google Search Console provides ongoing monitoring of your schema markup through the Enhancements section. The Structured Data report shows all pages where Google has detected markup, along with any errors or warnings. You can use this report to identify issues across your entire site and track the status of fixes over time.

Common Errors to Check

When testing schema markup, watch for these common errors: missing required properties, incorrect property values, invalid data types, nested item issues, and syntax errors in JSON-LD format. The testing tools will identify each of these issues with specific error messages that guide you toward the solution.

Testing Process

Follow these steps to test your schema markup: First, create your markup using a generator or manual coding. Second, paste your code into the Rich Results Test to check for basic eligibility. Third, fix any errors or warnings identified. Fourth, test again to confirm all issues are resolved. Fifth, add the markup to your live page. Sixth, test the live URL to confirm the markup is being detected correctly.

Testing for Different Schema Types

Different schema types have different testing requirements. Product schema needs the Rich Results Test to verify product rich result eligibility. Recipe schema needs the same test for recipe rich results. Some schema types like Organization may not produce visible rich results but should still be validated for correctness. Use the Schema Markup Validator for types that don't have specific rich result tests.

Automated Testing Tools

Several third-party tools offer automated schema markup testing. Screaming Frog SEO Spider can crawl your entire site and identify pages with schema markup issues. Merkle's Schema Markup Generator includes a built-in validator. SEMrush and Ahrefs include structured data audits in their site audit tools. These automated tools help you find and fix issues across large websites.

Regular Testing Schedule

Testing should not be a one-time activity. Google's rich result requirements can change, and your markup may break due to website updates. Establish a regular testing schedule to check your markup monthly or after any significant site changes. Search Console will alert you to new issues, but proactive testing helps catch problems before they affect your search visibility.
