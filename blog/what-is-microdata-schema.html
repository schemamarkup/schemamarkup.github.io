What is Microdata Schema?

June 2026 · 442 words

Microdata is an HTML specification that allows you to add structured data directly to your web page's visible content using HTML attributes. Unlike JSON-LD which uses a separate script block, Microdata integrates the structured data into the same HTML elements that display your content. This approach was one of the earliest methods for implementing schema markup and is still supported by all major search engines.

How Microdata Works

Microdata works by adding three main attributes to your HTML elements. The itemscope attribute declares that an HTML element contains a structured data item. The itemtype attribute specifies the Schema.org type for that item. The itemprop attribute identifies individual properties within the item. For example, a div with itemscope and itemtype attributes would contain spans with itemprop attributes for each property value.

Microdata Example

A typical Microdata implementation for an Organization would involve adding itemscope itemtype attributes to a div element, then using itemprop attributes on child elements to mark up the organization name, URL, and logo. The visible text content of those child elements becomes the property value. This tight integration between visible content and structured data is the defining characteristic of Microdata.

Advantages of Microdata

Microdata has some advantages in specific situations. Because the markup is embedded in your content, it naturally stays synchronized with your visible content. If you update the visible text, the structured data updates automatically. Microdata also works well for simple pages where you want to add a small amount of structured data without creating a separate script block.

Disadvantages of Microdata

Microdata has several significant disadvantages compared to JSON-LD. It clutters your HTML with many additional attributes, making your code harder to read and maintain. It is more difficult to debug because the structured data is spread throughout your HTML. It is harder to update because you need to find and modify individual HTML elements. It can also be limiting when you need to describe complex relationships that don't correspond neatly to your visible HTML structure.

Microdata vs. JSON-LD

Google recommends JSON-LD over Microdata for most use cases. JSON-LD is easier to implement, maintain, and debug. It doesn't clutter your HTML and it handles complex relationships more naturally. However, Microdata is still a valid format and is fully supported by Google. If you have an existing Microdata implementation, there is no urgent need to convert it, but new implementations should use JSON-LD.

When to Use Microdata

Microdata might be appropriate in certain situations. If you're working on a simple static site and want to add a small amount of schema markup quickly, Microdata can be a straightforward option. Some older CMS platforms and themes still use Microdata by default. However, for most new projects, JSON-LD is the better choice.
