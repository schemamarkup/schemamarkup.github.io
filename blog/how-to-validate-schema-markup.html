How do I validate Schema Markup?

June 2026 · 537 words

Validating schema markup is essential to ensure that search engines can process your structured data correctly. Validation checks your markup against the official Schema.org specification and Google's rich result guidelines. Proper validation prevents errors that could prevent your content from appearing in rich results.

What Is Schema Validation?

Schema validation is the process of checking your structured data against the official Schema.org vocabulary to ensure it is technically correct. Validation verifies that you are using valid types, valid properties for those types, and correct data formats for each property value. It also checks for missing required properties and syntax errors in your markup format.

Using Google's Rich Results Test

Google's Rich Results Test is the primary validation tool for schema markup. Enter a URL or paste your markup code to check it against Google's rich result requirements. The tool displays any errors or warnings, provides a preview of how your rich result might appear, and indicates which rich result types your page is eligible for. Pass this test before deploying any schema markup.

Using the Schema.org Validator

The official Schema.org validator provides comprehensive validation against the complete Schema.org vocabulary. It checks all types and properties, not just those that produce rich results. This validator is useful for complex markup with multiple schema types or less common schema types that may not be covered by Google's Rich Results Test.

Understanding Validation Errors

Validation errors fall into several categories. Syntax errors occur when your JSON-LD is not valid JSON, such as missing commas or braces. Type errors occur when you use an invalid type name or a property that doesn't belong to the specified type. Value errors occur when you provide a value in the wrong format, such as a string where a number is expected. Missing property errors occur when you omit required properties.

Fixing Common Validation Issues

Most validation issues are easy to fix. Syntax errors are corrected by reviewing your JSON structure for missing punctuation. Type errors are fixed by checking the Schema.org documentation for the correct type name. Value errors are resolved by converting the value to the expected data type. Missing properties are addressed by adding the required information to your markup.

Validation for Different Formats

JSON-LD, Microdata, and RDFa each have different validation requirements. JSON-LD requires valid JSON syntax and proper use of @context and @type. Microdata requires correct use of itemscope, itemtype, and itemprop attributes. RDFa requires proper use of typeof and property attributes. Use format-specific validators or the Schema.org validator which supports all three formats.

Automated Validation Tools

Several tools can automate schema validation across your entire website. Google Search Console provides ongoing monitoring through the Structured Data report. Screaming Frog SEO Spider can crawl your site and validate schema markup on every page. Merkle's Schema Markup Validator offers bulk validation. These tools help you maintain schema quality at scale.

Validation Best Practices

Follow these best practices for schema validation: Always validate markup before deploying to production. Test both the raw code and the live URL. Fix all errors and warnings, not just errors. Re-validate after making changes. Monitor Search Console for new issues. Keep your markup updated when content changes. Follow these practices to ensure your schema markup works correctly.
