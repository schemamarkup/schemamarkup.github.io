What is Event Schema?

June 2026 · 534 words

Event schema is a type of structured data that describes organized events such as concerts, conferences, workshops, webinars, festivals, sports games, and more. It provides properties for the event name, date, time, location, performers, organizers, and ticket information. When Google detects valid Event schema, it can display your event details directly in search results, including in the event-rich result format and Google's event search features.

Key Properties of Event Schema

Event schema includes essential properties for describing events. The name property is required and should clearly identify the event. The startDate and endDate properties specify when the event begins and ends in ISO 8601 format. The location property describes where the event takes place using Place or VirtualLocation schema. The offers property provides ticket information using Offer schema. The performer property identifies who is performing or presenting at the event.

Event Schema Subtypes

Schema.org defines numerous event subtypes for different types of events. BusinessEvent, ChildrensEvent, ComedyEvent, DanceEvent, EducationEvent, Festival, FoodEvent, LiteratureEvent, MusicEvent, SaleEvent, SocialEvent, SportsEvent, TheaterEvent, and VisualArtsEvent are all subtypes of Event. Each subtype inherits all Event properties and may add specialized properties. Using the correct subtype helps search engines categorize your event appropriately for event search features.

Online and Hybrid Events

Event schema supports online and hybrid events through the VirtualLocation schema. For online events, use the location property with VirtualLocation that includes a url property for the event access link. For hybrid events that have both in-person and online components, you can include multiple location entries. The eventAttendanceMode property specifies whether the event is offline, online, or mixed. These properties are essential for webinars, virtual conferences, and hybrid gatherings.

Event Status and Schedule

The eventStatus property indicates whether an event is scheduled, rescheduled, postponed, cancelled, or moved online. Keeping this property updated helps users find accurate information about event changes. The eventSchedule property supports recurring events by linking to a Schedule object that defines the recurrence pattern. This is useful for weekly classes, monthly meetups, or annual conferences. The schedule property helps users understand when recurring events take place.

Ticket and Pricing Information

The offers property uses Offer schema to describe ticket availability and pricing. Each offer can specify price, currency, availability, and a URL for purchasing tickets. You can include multiple offers for different ticket types like general admission, VIP, and early bird pricing. The validFrom property for each offer indicates when ticket sales begin. Comprehensive ticket markup helps users find pricing and purchase options directly from search results.

Event Organization and Performers

The organizer property identifies the person or organization responsible for planning the event. The performer property lists who will be performing or speaking at the event. Both properties can use Person or Organization schema to provide detailed information. For multi-performer events like music festivals, include all major performers. This information helps search engines display event details and can attract attendees who follow specific performers or organizers.

Conclusion

Event schema is essential for any website promoting events. It enables rich search results with dates, locations, and ticket information, making it easier for users to find and attend your events.
